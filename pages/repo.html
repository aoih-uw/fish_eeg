Repository Information

Background

This project analyzes auditory evoked potentials (AEPs) in Hydrolagus colliei (spotted ratfish) using a multi-stage electrophysiological pipeline. The analysis focuses on detecting the double-frequency (2f) neural response to acoustic stimuli using ICA-based denoising, bootstrapping, and SNR calculations.

Understanding how fish respond to auditory stimuli is crucial for marine conservation, especially in relation to anthropogenic noise pollution in environments like the Puget Sound. This research provides insights into the auditory capabilities of cartilaginous fish and their potential vulnerability to underwater acoustic disturbances.

Intended Audience

  • Researchers who need to analyze auditory fish EEG recordings
  • Collaborators of researchers who want to understand the data analysis pipeline and its results
  • Generally scientifically minded curious people who want to know about what sounds fish can hear

Installation

This package provides a complete workflow for processing, analyzing, and visualizing auditory fish EEG recordings. It includes tools for data ingestion, preprocessing, feature extraction, statistical analysis, and interactive web-based visualization.

Option 1: Install directly from GitHub

pip install git+https://github.com/aoih-uw/fish_eeg.git

Option 2: Clone the repository and install locally

git clone https://github.com/aoih-uw/fish_eeg.git
cd fish_eeg
pip install .

These commands will build and install the package from pyproject.toml

Example Usage

1. Ensure your dataset is in the right format (.mat file with the following structure):

f = h5py.File('fish01.mat', 'r')
ex_data = f['ex']
channel_keys = ['ch1','ch2','ch3','ch4']
period_keys = ['prestim_sig','stimresp_sig','poststimresp_sig']

2. Place your dataset in the data/ directory

3. Run the analysis pipeline:

python pipeline.py --input data/'fish01.mat' --output results/

Data Sources

1. Fish EEG Recordings

Collected by Aoi Hunsaker in June 2025. Multi-channel electrophysiological recordings from 7 subjects at the Sisneros Lab, University of Washington.

2. Passive Acoustic Data

Environmental noise data from Puget Sound via NOAA's Google Cloud Storage (Orcasound Labs). Used to contextualize auditory thresholds against real-world underwater noise levels.

Dataset Details: Multi-channel electrophysiological recordings from 7 subjects:

  • deadFish (control/baseline)
  • hydrolagusColliei_4 through hydrolagusColliei_9 (6 test subjects)

Data Format: MATLAB .mat files containing 4-channel electrode recordings organized by stimulus frequency (Hz) and amplitude (dB). Each recording includes multiple trials across channels ch1-ch4.

Collection Method: Auditory stimuli were presented at various frequencies (55-385 Hz) and amplitudes (70-150 dB), with neural responses recorded via surface electrodes.

Limitations: Some frequencies (50 Hz, 385 Hz) showed artifacts and were filtered out. Certain amplitude ranges (145-150 dB) contained problematic data. Subject hydrolagusColliei_7 had issues at 55 Hz and 100 Hz. Sample size is limited to 6 live subjects plus one control.

Software Architecture

System Components:

  • Data Ingestion Manager (preprocess.py): Loads .mat EEG datasets, validates file structure, performs artifact rejection, and prepares data for downstream analysis. Outputs structured data objects containing raw EEG signals and metadata.
  • EEG Data Filtering (filters.py): Performs bandpass filtering (70-1400 Hz) to remove irrelevant signals from analysis. Takes filter parameters (low/high cutoff frequencies, order) and outputs filtered EEG datasets.
  • ICA Denoising Manager (denoiser.py, reconstruct.py): Performs ICA-based denoising to remove irrelevant signals. Outputs denoised EEG datasets and ICA decomposition objects (unmixing matrix, time-series components).
  • Statistical Analysis Manager (statistics.py): Performs bootstrap-based confidence interval tests (100 iterations) on cleaned and filtered EEG datasets. Outputs results determining if the fish heard a given sound frequency.
  • Visualization Manager (plotting.py): Plots results of statistical analysis using Altair library. Generates interactive HTML plots with tooltips showing auditory sensitivity across tested frequencies.
  • Web Dashboard (index.html): Multi-page HTML interface with responsive navigation, search functionality, guided tour, and embedded interactive visualizations.

Data Flow:

Raw .mat files ‚Üí preprocess.py (artifact removal) ‚Üí filters.py (bandpass filter) ‚Üí denoiser.py (ICA denoising) ‚Üí Period separation (prestim/stimresp) ‚Üí statistics.py (FFT ‚Üí Bootstrap ‚Üí Confidence intervals) ‚Üí plotting.py (interactive visualizations) ‚Üí index.html (web dashboard)

Lessons Learned & Future Work

Software Engineering Lessons:

  • Modular Design: Separating preprocessing, analysis, and visualization into distinct modules improved code maintainability and testing
  • Environment Management: Using Conda environments ensured reproducibility across different systems and prevented dependency conflicts
  • Documentation: Extensive README files and inline comments were crucial for onboarding new lab members and future users
  • Version Control: Git workflow with .gitignore for large data files kept the repository clean and collaborative
  • Interactive Visualization: Using Altair for web-based interactive plots made results more accessible than static images
  • Data Organization: Structured CSV outputs and consistent naming conventions simplified downstream analysis

Future Work:

  • Expand to more subjects and species for comparative analysis
  • Implement real-time filtering and audio playback of stimuli on the website
  • Add machine learning classification of response patterns
  • Develop automated pipeline for processing new recordings
  • Create API for programmatic access to processed data
  • Add statistical power analysis tools for experimental design
