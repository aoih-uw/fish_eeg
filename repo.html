Repository Information

Background

This project analyzes auditory evoked potentials (AEPs) in Hydrolagus colliei (spotted ratfish) using a multi-stage electrophysiological pipeline. The analysis focuses on detecting the double-frequency (2f) neural response to acoustic stimuli using ICA-based denoising, bootstrapping, and SNR calculations.

Understanding how fish respond to auditory stimuli is crucial for marine conservation, especially in relation to anthropogenic noise pollution in environments like the Puget Sound. This research provides insights into the auditory capabilities of cartilaginous fish and their potential vulnerability to underwater acoustic disturbances.

Data Used

Dataset: Multi-channel electrophysiological recordings from 7 subjects:

  • deadFish (control/baseline)
  • hydrolagusColliei_4 through hydrolagusColliei_9 (6 test subjects)

Data Format: MATLAB .mat files containing 4-channel electrode recordings organized by stimulus frequency (Hz) and amplitude (dB). Each recording includes multiple trials across channels ch1-ch4.

Collection Method: Auditory stimuli were presented at various frequencies (55-385 Hz) and amplitudes (70-150 dB), with neural responses recorded via surface electrodes.

Limitations: Some frequencies (50 Hz, 385 Hz) showed artifacts and were filtered out. Certain amplitude ranges (145-150 dB) contained problematic data. Subject hydrolagusColliei_7 had issues at 55 Hz and 100 Hz. Sample size is limited to 6 live subjects plus one control.

All visualizations are created using Altair and are fully interactive in the browser.

Design

System Components:

  • Data Ingestion: Python scripts read MATLAB .mat files and organize trials by frequency/amplitude
  • Preprocessing Module: Artifact rejection based on RMS thresholds, trial balancing across channels
  • Signal Processing: Bandpass filtering (70-1400 Hz), ICA denoising with component selection
  • Statistical Analysis: Bootstrap resampling (100 iterations), FFT analysis, SNR calculation at 2f
  • Visualization Engine: Altair library generates interactive HTML plots with faceting by frequency
  • Web Dashboard: Multi-page HTML interface with responsive navigation, search functionality, and embedded visualizations

Lessons Learned & Future Work

Software Engineering Lessons:

  • Modular Design: Separating preprocessing, analysis, and visualization into distinct modules improved code maintainability and testing
  • Environment Management: Using Conda environments ensured reproducibility across different systems and prevented dependency conflicts
  • Documentation: Extensive README files and inline comments were crucial for onboarding new lab members and future users
  • Version Control: Git workflow with .gitignore for large data files kept the repository clean and collaborative
  • Interactive Visualization: Using Altair for web-based interactive plots made results more accessible than static images
  • Data Organization: Structured CSV outputs and consistent naming conventions simplified downstream analysis

Future Work:

  • Expand to more subjects and species for comparative analysis
  • Implement real-time filtering and audio playback of stimuli on the website
  • Add machine learning classification of response patterns
  • Develop automated pipeline for processing new recordings
  • Create API for programmatic access to processed data
  • Add statistical power analysis tools for experimental design
